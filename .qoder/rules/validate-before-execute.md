# 执行前校验规则

## 规则

对于用户的每一次输入，在执行任何操作之前，必须先完成以下检查：

1. **合理性检查** — 请求是否在当前项目上下文中合理（结构、命名、技术栈等）
2. **一致性检查** — 是否与已有内容、路径、命名规范或此前的决策存在矛盾
3. **歧义确认** — 如果发现任何不清晰、矛盾或不一致之处，**先向用户确认，再执行**
4. **对齐后执行** — 只有在双方达成一致后，才开始实际操作

## 不允许的行为

- 在输入存在歧义时直接假设用户意图并执行
- 在发现路径/命名/结构不一致时默默修正而不告知用户
